# 品牌更名与去登录轻量设计

## 目标

将软件对外显示名称统一改为“AI智能编程”，删除首页指定宣传区块，并取消登录要求。

## 边界

- 删除首页以下区块：
  - SenseCraft AI 应用中心
  - 用户社区
  - 项目赞助者
- 删除或隐藏登录入口、登录弹窗、用户中心入口
- 取消前端登录门禁，让原本需要登录才能进入的功能直接可用
- 保留 `AuthService` 与自定义模型 API 相关代码，不做底层鉴权逻辑重构

## 风险

- 登录相关逻辑分散在多个组件里，若只删 UI 不删门禁，可能仍会弹“请先登录”
- 若误删 `AuthService` 依赖链，可能影响与模型服务相关的已有能力

## 执行策略

采用“保留底层、移除入口、解除门禁”的最小改动方案：

1. 统一替换品牌显示名
2. 删除首页指定区块
3. 隐藏登录入口与用户中心入口
4. 将前端 `requireLogin()` 等门禁改为直接放行
5. 保留认证服务与模型相关代码，避免误伤自定义模型 API

## 验证方式

- 应用标题、首页标题、打包名称显示为“AI智能编程”
- 首页不再显示指定 3 个区块
- 界面中不再出现登录入口或用户中心入口
- 原先受登录限制的相关入口可直接使用
